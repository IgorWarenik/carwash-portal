## 1. Общая архитектура: Hub-and-Spoke

Весь сайт строится по модели «хаб и спицы». Хаб-страницы аккумулируют ссылочный вес и передают его спицам (детальным страницам). Спицы возвращают сигналы авторитета хабам через хлебные крошки и контекстные ссылки.

```
Главная (/)
    |
    ├── /catalog/         ← хаб справочника
    │       ├── /catalog/moscow/           ← городской хаб
    │       │       ├── /catalog/moscow/contactnaya/     ← программная страница
    │       │       └── /catalog/avto-blessk-taganka/    ← детальная карточка
    │
    ├── /invest/          ← инвестиционный хаб
    │       ├── /invest/open/              ← подхаб «открыть»
    │       ├── /invest/buy/               ← подхаб «купить»
    │
    ├── /suppliers/       ← хаб поставщиков
    ├── /franchises/      ← хаб франшиз
    ├── /tools/           ← хаб инструментов
    └── /guides/          ← медиахаб
```

## 2. Программный SEO: матрица город × тип

### Приоритетные кластеры

**Кластер 1: Справочный (Directory)**
Паттерн: `автомойки {city}`, `автомойки {type} {city}`, `автомойка {type} {district}`

Матрица генерации страниц:
- 50 городов × 5 типов = **250 страниц** (Tier A)
- 50 городов × 10 районов (только топ-10 городов) = **500 страниц** (Tier B)
- Итого: ~750 программных страниц в каталоге

**Кластер 2: Инвестиционный (Investment)**
Паттерн: `открыть автомойку {city}`, `купить автомойку {city}`, `стоимость открытия автомойки {city}`

- 20 городов × 3 intent = **60 страниц**

**Кластер 3: Поставщики (Suppliers)**
Паттерн: `{category} для автомоек`, `{product} для автомойки купить`

- 10 категорий × 5 продуктов = **50 страниц**

### Минимальный объём контента для программных страниц

| Элемент | Минимум |
|---|---|
| Уникальный H1 | Да |
| Вводный абзац (≥100 слов, уникальный) | Да |
| Фактические данные (кол-во объектов, адреса) | Да |
| Блок FAQ (3–5 вопросов) | Да |
| Внутренние ссылки | ≥3 |
| Schema.org разметка | Да |

### Правило против тонкого контента
Страница НЕ публикуется, если:
- Количество автомоек в каталоге < 3 для данной комбинации
- Уникальный текст < 150 слов
- Нет реальных данных (только шаблонный текст)

Такие страницы получают статус `draft` и не включаются в sitemap.

## 5. Стратегия sitemap

### Структура sitemap
```
/sitemap.xml                   ← индексный sitemap
/sitemap-static.xml            ← статические страницы (хабы, about, tools)
/sitemap-catalog.xml           ← каталог автомоек (динамический)
/sitemap-city-type.xml         ← программные страницы город×тип
/sitemap-suppliers.xml         ← поставщики и франшизы
/sitemap-invest.xml            ← инвестиционный центр
/sitemap-guides.xml            ← гайды и статьи
/sitemap-glossary.xml          ← глоссарий
```

### Правила sitemap
- Только страницы со статусом `published`
- Только страницы с уникальным контентом (не noindex)
- `<changefreq>` по типу страниц:
  - Главная, хабы: `weekly`
  - Карточки автомоек: `monthly`
  - Гайды/статьи: `monthly` (при обновлении — `weekly`)
  - Программные страницы: `monthly`
- `<priority>`:
  - Главная: `1.0`
  - Хабы Tier 1: `0.9`
  - Городские хабы: `0.8`
  - Детальные страницы: `0.6`
  - Гайды: `0.7`
  - Глоссарий: `0.5`
- `<lastmod>`: актуальная дата последнего изменения из БД

### Robots.txt
```
User-agent: *
Allow: /

Disallow: /admin/
Disallow: /api/
Disallow: /invest/sell/submit/
Disallow: /*?*  (параметры фильтрации)

Sitemap: https://portal.ru/sitemap.xml
```

## 6. Каноникалы

### Правила canonical

| Ситуация | Canonical |
|---|---|
| Страница пагинации `/catalog/moscow/?page=2` | `/catalog/moscow/` |
| Страница с параметрами фильтра `?type=contact` | `/catalog/moscow/contactnaya/` (или self если нет отдельной страницы) |
| Дублирующийся контент (парсинговый импорт) | На оригинальный URL |
| AMP-версия (если будет) | На основной URL |
| Все уникальные страницы | Self-canonical |

### Реализация
- `<link rel="canonical" href="..." />` в `<head>`
- Генерация через `packages/seo` утилиту `getCanonicalUrl()`
- Для программных страниц — автоматическая генерация по паттерну

## 7. Антиправила тонкого контента

### Критерии noindex

Страница получает `noindex, follow` если:
1. Количество объектов в листинге = 0
2. Основной текст < 150 слов И нет уникальных данных
3. Страница создана автоматически без ручного одобрения редактора
4. Страница является точным дублём другой страницы

### Критерии удаления из sitemap (но не noindex)

1. Страница город × тип с < 3 автомойками
2. Статья в статусе `draft`
3. Страница поставщика без заполненного описания

### Программа обогащения контента

Для программных страниц с риском тонкого контента обязательно наличие:
- Уникального вводного абзаца (шаблон + переменные = уникализация)
- Реального количества объектов с динамическим обновлением
- FAQ секции (3 вопроса, генерируются из шаблона + данных города)
- Блока «Полезное по теме» (3 внутренние ссылки на гайды)

## 8. Воркфлоу обновления контента (Freshness)

### Сигналы свежести для Яндекса

1. **`dateModified` в JSON-LD** — обновляется при любом изменении страницы
2. **`<lastmod>` в sitemap** — синхронизировано с БД
3. **Динамические блоки** — «Последние добавленные автомойки», «Новые объявления» обновляются в реальном времени

### Расписание обновлений

| Тип контента | Минимальная частота | Ответственный |
|---|---|---|
| Данные автомоек (часы, адрес) | Ежеквартально | Редактор / data-ingestion агент |
| Гайды/статьи | 6–12 месяцев | Редактор |
| Программные страницы (текст) | Ежегодно | SEO-агент |
| Аналитика рынка | Ежеквартально | Редактор |
| Листинги купли-продажи | Реальное время | Автоматически |

## 9. Семантическое ядро: приоритеты

### Кластер 1: Справочные запросы (высокий объём)
- `автомойки [город]` — ВЧ, коммерческая
- `автомойка рядом` — геозависимый
- `контактная/бесконтактная/самообслуживание автомойка [город]`
- `автомойка [район города]`

### Кластер 2: Инвестиционные запросы (средний объём, высокая монетизация)
- `как открыть автомойку` — НЧ/СЧ, информационный
- `открыть автомойку самообслуживания` — точный
- `сколько стоит открыть автомойку` — коммерческо-информационный
- `купить готовую автомойку` — ВЧ, коммерческий
- `купить автомойку [город]`
- `франшиза автомойки` — ВЧ
- `автомойка самообслуживания франшиза`

### Кластер 3: B2B запросы (НЧ, высокий CPL)
- `оборудование для автомойки купить`
- `химия для автомоек оптом`
- `портальная мойка цена`
- `туннельная автомойка оборудование`

### Кластер 4: Информационные запросы (E-E-A-T)
- `срок окупаемости автомойки`
- `рентабельность автомойки`
- `бизнес план автомойки`
- `документы для открытия автомойки`
- `персонал для автомойки`

### Приоритет индексации
1. Главная + хабы Tier 1
2. Топ-10 городских хабов
3. Страницы Москва × все типы
4. Коммерческие гайды (открыть/купить)
5. Программные страницы топ-20 городов
6. Детальные карточки автомоек
7. Глоссарий и справочные страницы
