# Skill: create-comparison-page

Создаёт страницу сравнения типов автомоек или форматов бизнеса.

## Шаги

1. **Определить тип сравнения**
   - `types`: сравнение типов моек (самообслуживание vs автоматическая vs ручная)
   - `cities`: сравнение рынков в разных городах
   - `formats`: сравнение форматов бизнеса (отдельно стоящая vs встроенная)

2. **Создать страницу**
   - URL паттерн: `/sravnenie/<type1>-vs-<type2>` или `/sravnenie/<city1>-vs-<city2>`
   - Компонент: `apps/web/app/(public)/sravnenie/[slug]/page.tsx`
   - Данные: из `packages/content/src/comparisons/` или из БД (benchmarks)

3. **Структура страницы**
   - H1: "Сравнение <X> и <Y>"
   - Таблица сравнения (ComparisonTable компонент)
   - Разделы для каждого варианта
   - Итоговые рекомендации
   - CTA: лид-форма или калькулятор

4. **SEO**
   - Title: `<X> vs <Y> — сравнение, плюсы и минусы | АвтомойкиПортал`
   - Structured data: `FAQPage` для типичных вопросов о сравнении
   - Canonical: абсолютный URL
   - Breadcrumbs: Главная → Сравнения → <название>

5. **Таблица сравнения**
   - Компонент: `apps/web/components/compare/ComparisonTable.tsx`
   - Данные из `packages/content/src/comparisons/<slug>.ts`
   - Параметры: инвестиции, окупаемость, риски, масштабируемость

6. **Перелинковка**
   - Ссылки на детальные страницы каждого варианта
   - Ссылка на калькулятор окупаемости
   - Ссылка на листинги в продаже

## Чеклист

- [ ] Страница доступна по /sravnenie/<slug>
- [ ] Таблица сравнения заполнена
- [ ] SEO метаданные уникальны
- [ ] JSON-LD FAQPage валиден
- [ ] Breadcrumbs присутствуют
- [ ] CTA ведёт к конверсии
- [ ] Перелинковка настроена
